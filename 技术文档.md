## 系统架构

程序采用经典的三层架构，各层职责清晰，互不耦合：

```
┌─────────────────────────────────────────────────────────┐
│                      UI 层（ui/）                        │
│  app.py  home  import  exam  study  wrong_book  history  │
│  负责：界面渲染、用户交互、事件响应                       │
└──────────────────────┬──────────────────────────────────┘
                       │ 调用
┌──────────────────────▼──────────────────────────────────┐
│                    业务逻辑层（core/）                    │
│         ExamEngine  StudyEngine  StatsEngine             │
│  负责：出题逻辑、计时评分、统计分析                       │
└──────────────────────┬──────────────────────────────────┘
                       │ 调用
┌──────────────────────▼──────────────────────────────────┐
│                   数据访问层（database/）                 │
│              DBManager（单例）  models                   │
│  负责：SQLite 连接管理、SQL 执行、数据模型定义            │
└──────────────────────┬──────────────────────────────────┘
                       │ 读写
┌──────────────────────▼──────────────────────────────────┐
│                   SQLite 数据库文件                       │
│                      exam.db                             │
└─────────────────────────────────────────────────────────┘

另有独立的导入模块（importer/）：
  文件（.xlsx/.docx）→ XlsxImporter/DocxImporter → BaseImporter.save_to_db() → DBManager
```

## 模块说明

### 入口与配置

#### `main.py` — 程序入口

职责：
1. 调用 `setup_logging()` 配置日志系统（同时输出到文件和控制台）
2. 调用 `DBManager.initialize()` 初始化数据库（建表、写默认设置）
3. 延迟导入并实例化 `App`，启动 tkinter 主循环

关键设计：延迟导入 `App`（在 `main()` 函数内部 `from ui.app import App`），确保数据库初始化完成后再触发 UI 相关代码。

---

#### `config.py` — 全局配置常量

职责：集中管理所有可调参数，避免魔法数字散落在各处。

| 常量 | 说明 |
|------|------|
| `APP_DATA_DIR` | 数据目录（开发模式：`data/`；打包后：`%APPDATA%\ExamClient\`） |
| `DB_PATH` | SQLite 数据库文件路径 |
| `LOG_PATH` | 日志文件路径 |
| `WINDOW_SIZE` | 初始窗口尺寸（`1100x720`） |
| `NAV_WIDTH` | 左侧导航栏宽度（`150` px） |
| `FONT_FAMILY` | 字体族（`Microsoft YaHei`，macOS 自动回退） |
| `COLOR_*` | 所有 UI 颜色常量（十六进制） |
| `DEFAULT_QUESTION_COUNT` | 默认出题数量（`50`） |
| `DEFAULT_TIME_LIMIT_MIN` | 默认时间限制（`60` 分钟） |

`get_app_data_dir()` 函数通过 `getattr(sys, 'frozen', False)` 判断是否处于 PyInstaller 打包环境，从而选择不同的数据目录。

### 数据访问层

#### `database/db_manager.py` — 数据库管理器

设计模式：**单例模式（Singleton Pattern）**

```
程序启动
    ↓
DBManager.initialize(db_path)   ← 只调用一次
    ↓
DBManager._instance = DBManager(db_path)
    ↓
_connect() → _create_tables() → _seed_settings()
    ↓
任意模块：DBManager.get_instance()  ← 获取同一个实例
```

核心方法：

| 方法 | 说明 |
|------|------|
| `initialize(db_path)` | 类方法，初始化单例，幂等（多次调用安全） |
| `get_instance()` | 类方法，获取单例，未初始化时抛出 RuntimeError |
| `execute(sql, params)` | 执行写操作（INSERT/UPDATE/DELETE），自动提交 |
| `executemany(sql, params_list)` | 批量写操作，一次提交 |
| `fetchall(sql, params)` | 查询多行，返回 `List[sqlite3.Row]` |
| `fetchone(sql, params)` | 查询单行，返回 `sqlite3.Row` 或 `None` |
| `get_setting(key, default)` | 读取 app_settings 表中的配置值 |
| `set_setting(key, value)` | 写入或更新配置值（INSERT OR REPLACE） |

**安全性说明：** 所有 SQL 均使用参数化查询（`?` 占位符），防止 SQL 注入。绝对不要用字符串拼接构造 SQL。

---

#### `database/models.py` — 数据模型

使用 Python `dataclass`（需 3.7+）将数据库行映射为强类型对象。

| 类 | 对应表 | 说明 |
|----|--------|------|
| `QuestionBank` | `question_banks` | 题库元数据 |
| `Question` | `questions` | 题目（含 `get_options()`、`check_answer()` 方法） |
| `ExamRecord` | `exams` | 考试记录 |
| `ExamAnswer` | `exam_answers` | 单题作答记录 |
| `WrongAnswerEntry` | `wrong_answers` | 错题本条目 |
| `ExamResult` | 无（内存对象） | 考试结束后的汇总结果 |

每个类都有 `from_row(cls, row: sqlite3.Row)` 类方法，从数据库查询结果构造对象。

`Question.check_answer(user_answer)` 的判题逻辑：将正确答案和用户答案都转为大写字符集合后比较，因此多选题答案顺序无关（`'AB' == 'BA'`）。

### 业务逻辑层

#### `core/exam_engine.py` — 考试引擎

使用流程：

```python
engine = ExamEngine(bank_id=1, question_count=20, time_limit=3600)
paper  = engine.generate_paper()   # 随机抽题
engine.start()                      # 记录开始时间
engine.submit_answer(q_id, 'B')    # 提交答案（每题调用一次）
result = engine.finish()            # 结束考试，写入数据库，返回 ExamResult
```

`generate_paper()` 支持两种模式：
- **普通模式**：按 `bank_id`、`q_types`、`categories` 筛选后随机抽取
- **错题练习模式**：`wrong_q_ids` 不为 None 时，直接按 ID 列表出题

`finish()` 执行的数据库操作：
1. INSERT INTO exams（写考试记录）
2. executemany INSERT INTO exam_answers（批量写答题明细）
3. 逐题更新 wrong_answers（新错题 INSERT，已有错题 UPDATE 计数）

---

#### `core/study_engine.py` — 学习模式引擎

维护一个题目列表和当前索引，提供 `next()`、`prev()`、`jump_to(index)` 导航方法。支持按分类筛选（`reload(category)`）。

---

#### `core/stats_engine.py` — 统计分析引擎

| 方法 | 说明 |
|------|------|
| `get_exam_history(limit)` | 最近 N 条考试记录（LEFT JOIN 题库名） |
| `get_exam_detail(exam_id)` | 单次考试的完整答题明细 |
| `get_score_trend(limit)` | 得分趋势数据（日期+分数，升序） |
| `get_wrong_questions(hide_mastered)` | 错题本列表（JOIN 题目信息） |
| `get_overview()` | 首页仪表盘汇总数据 |
| `mark_mastered(question_id)` | 标记已掌握 |
| `delete_wrong_entry(question_id)` | 从错题本删除 |

### 导入模块

#### `importer/base_importer.py` — 导入器基类

设计模式：**模板方法模式（Template Method Pattern）**

```
BaseImporter（抽象类）
    ├── parse()          ← 抽象方法，子类实现
    ├── validate_question(q)  ← 基类实现，校验单题
    └── save_to_db(questions) ← 基类实现，写入数据库

XlsxImporter(BaseImporter)
    └── parse()  ← 解析 Excel

DocxImporter(BaseImporter)
    └── parse()  ← 解析 Word
```

`save_to_db()` 的事务逻辑：
1. 先插入 `question_banks` 记录，获取 `bank_id`
2. 逐题校验 → 写入 `questions`
3. 更新 `question_banks.question_count`
4. 若成功数为 0，删除空题库记录（避免留下无用数据）

---

#### `importer/xlsx_importer.py` — Excel 解析器

- 使用 `openpyxl` 以只读模式打开文件（`read_only=True`），节省内存
- 支持多 Sheet，Sheet 名作为默认分类
- `_guess_type()` 方法：当题型列无法识别时，根据答案内容猜测题型

---

#### `importer/docx_importer.py` — Word 解析器

- 使用 `python-docx` 提取所有段落文本
- 状态机解析：遇到 `【题型标记】` 切换题型，遇到空行分割题目块
- 正则表达式识别选项行（`A.` `A、` `A）` 等格式）、答案行、解析行

---

### UI 层

#### `ui/app.py` — 主窗口

- 继承 `tk.Tk`，是整个 GUI 的根窗口
- 构建左侧导航栏（固定宽度 150px）+ 右侧内容区
- `_register_frames()`：延迟导入并实例化所有页面，用 `place(relx=0, rely=0, relwidth=1, relheight=1)` 叠放，通过 `tkraise()` 切换显示
- `show_frame(name, **kwargs)`：切换页面，调用旧页面 `on_hide()`、新页面 `on_show(**kwargs)`

---

#### `ui/base_frame.py` — 页面基类

所有页面继承此类，提供：
- `on_show(**kwargs)` / `on_hide()`：生命周期钩子
- `make_label()` / `make_button()` / `make_card()`：统一样式的组件工厂方法
- `format_duration(seconds)`：秒数格式化为 `mm:ss` 或 `hh:mm:ss`
- `_darken(hex_color)`：颜色加深（用于按钮悬停效果）

## 数据库结构

### `question_banks` — 题库元数据

```sql
CREATE TABLE question_banks (
    id             INTEGER PRIMARY KEY AUTOINCREMENT,
    name           TEXT    NOT NULL,          -- 用户自定义名称
    source_file    TEXT    NOT NULL,          -- 原始文件名
    file_type      TEXT    NOT NULL,          -- 'xlsx' 或 'docx'
    imported_at    TEXT    NOT NULL,          -- ISO8601 时间字符串
    question_count INTEGER DEFAULT 0,         -- 实际题目数量
    is_active      INTEGER DEFAULT 1          -- 1=正常，0=软删除
)
```

### `questions` — 题目

```sql
CREATE TABLE questions (
    id           INTEGER PRIMARY KEY AUTOINCREMENT,
    bank_id      INTEGER NOT NULL REFERENCES question_banks(id),
    q_type       TEXT    NOT NULL,   -- 'single'|'multi'|'truefalse'|'scenario'
    content      TEXT    NOT NULL,   -- 题干
    option_a     TEXT,               -- 选项 A（判断题为 NULL）
    option_b     TEXT,
    option_c     TEXT,
    option_d     TEXT,
    option_e     TEXT,
    correct_ans  TEXT    NOT NULL,   -- 'A'|'AB'|'True'|'False'
    explanation  TEXT,               -- 解析（可为空）
    category     TEXT    DEFAULT '', -- 分类标签
    difficulty   INTEGER DEFAULT 2,  -- 1=易 2=中 3=难
    created_at   TEXT    NOT NULL
)
```

### `exams` — 考试记录

```sql
CREATE TABLE exams (
    id            INTEGER PRIMARY KEY AUTOINCREMENT,
    bank_id       INTEGER REFERENCES question_banks(id),  -- 可为 NULL（错题练习）
    total_q       INTEGER NOT NULL,
    score         REAL    NOT NULL,   -- 百分制，如 85.5
    correct_count INTEGER NOT NULL,
    wrong_count   INTEGER NOT NULL,
    duration_sec  INTEGER NOT NULL,   -- 用时（秒）
    time_limit    INTEGER,            -- 设定时限（秒），NULL=无限制
    started_at    TEXT    NOT NULL,
    finished_at   TEXT    NOT NULL
)
```

### `exam_answers` — 答题明细

```sql
CREATE TABLE exam_answers (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    exam_id     INTEGER NOT NULL REFERENCES exams(id),
    question_id INTEGER NOT NULL REFERENCES questions(id),
    user_answer TEXT    NOT NULL,   -- 用户答案，空字符串=未作答
    is_correct  INTEGER NOT NULL,   -- 1=正确 0=错误
    time_spent  INTEGER DEFAULT 0   -- 该题用时（秒）
)
```

### `wrong_answers` — 错题本

```sql
CREATE TABLE wrong_answers (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    question_id INTEGER NOT NULL REFERENCES questions(id) UNIQUE,
    wrong_count INTEGER DEFAULT 1,   -- 累计答错次数
    last_wrong  TEXT    NOT NULL,    -- 最近答错时间
    last_answer TEXT    NOT NULL,    -- 最近提交的错误答案
    is_mastered INTEGER DEFAULT 0    -- 0=待复习 1=已掌握
)
```

### `app_settings` — 应用设置

```sql
CREATE TABLE app_settings (
    key   TEXT PRIMARY KEY,
    value TEXT NOT NULL
)
-- 默认值：theme=light, font_size=11,
--         default_time_limit=60, default_question_count=50
```

## 数据流说明

### 导入题库

```
用户选择文件
    ↓
ImportFrame._start_import()
    ↓ 后台线程（threading.Thread）
XlsxImporter(filepath, bank_name) 或 DocxImporter(filepath, bank_name)
    ↓
importer.parse()          → 返回 List[Dict]（题目字典列表）
    ↓
importer.save_to_db()     → 写入 question_banks + questions 表
    ↓
通过 after() 回调更新 UI（线程安全）
```

### 考试流程

```
ExamSetupFrame → ExamEngine(bank_id, count, time_limit, q_types)
    ↓
engine.generate_paper()   → SELECT FROM questions（随机抽题）
    ↓
ExamFrame（答题界面）
    ↓ 每题
engine.submit_answer(q_id, answer)
    ↓ 交卷
engine.finish()
    → INSERT INTO exams
    → INSERT INTO exam_answers（批量）
    → UPDATE/INSERT wrong_answers
    → 返回 ExamResult
    ↓
ExamResultFrame（结果展示）
```

## 打包原理

PyInstaller 将 Python 解释器、所有依赖库、项目代码打包为单个可执行文件。

### build.spec 关键配置说明

```python
a = Analysis(
    ['main.py'],          # 入口文件
    datas=[('assets', 'assets')],  # 将 assets 目录打包进去
    hiddenimports=[       # PyInstaller 无法自动发现的隐式导入
        'openpyxl.cell._writer',
        'openpyxl.styles.stylesheet',
        'docx', 'docx.oxml', 'docx.oxml.ns',
    ],
    excludes=['matplotlib', 'numpy', 'PIL', 'scipy', 'pandas', 'pytest'],
    # 排除不需要的大型库，减小体积
)

exe = EXE(
    ...
    console=False,   # 不显示黑色命令行窗口
    upx=True,        # 启用 UPX 压缩（需系统安装 upx，否则自动跳过）
)
```

### 打包后的路径处理

`config.py` 中的 `get_app_data_dir()` 通过 `getattr(sys, 'frozen', False)` 判断运行环境：

```python
if getattr(sys, 'frozen', False):
    # 打包后：数据存在 %APPDATA%\ExamClient\
    base = os.environ.get('APPDATA', os.path.expanduser('~'))
    path = os.path.join(base, 'ExamClient')
else:
    # 开发模式：数据存在项目目录的 data/ 下
    path = os.path.join(os.path.dirname(os.path.abspath(__file__)), 'data')
```

这确保了打包后的程序将数据写入用户目录，而不是程序所在目录（后者在某些系统上没有写权限）。

## Python 版本兼容性

| 特性 | 最低版本 | 本项目使用情况 |
|------|---------|--------------|
| `dataclass` | 3.7 | 已使用（`database/models.py`） |
| f-string | 3.6 | 已使用 |
| `typing.Optional/List/Dict` | 3.5 | 已使用 |
| `abc.ABC` / `abstractmethod` | 3.4 | 已使用 |
| 内置泛型 `list[str]`、`dict[str,int]` | 3.9 | **未使用**，安全 |
| walrus operator `:=` | 3.8 | **未使用**，安全 |
| `match/case` | 3.10 | **未使用**，安全 |

**结论：最低要求 Python 3.7，推荐 3.10 或 3.11。**

Python 3.6 不支持（缺少 `dataclass`）。Python 3.13 可运行但部分旧版依赖需升级（见 requirements.txt）。

## 错误排查

### 启动报错：`ModuleNotFoundError: No module named 'openpyxl'`

原因：依赖未安装到当前 Python 环境。

解决：
```bat
:: Windows
.venv\Scripts\pip install -r requirements.txt
```
```bash
# macOS
.venv313/bin/pip install -r requirements.txt
```

---

### 启动报错：`RuntimeError: DBManager 尚未初始化`

原因：`DBManager.initialize()` 未被调用就尝试使用数据库。通常是直接运行了 UI 模块而不是 `main.py`。

解决：始终通过 `python main.py` 启动程序。

---

### 导入报错：`成功导入 0 题`

原因：所有题目都未通过 `validate_question()` 校验。

排查步骤：
1. 查看导入日志中的具体跳过原因
2. 常见问题：答案列为空、题型列填写错误、Excel 列顺序不对

---

### 打包报错：`ModuleNotFoundError` 在打包后运行时出现

原因：某个模块被 PyInstaller 遗漏，未打包进去。

解决：在 `build.spec` 的 `hiddenimports` 列表中添加该模块名，重新打包。

---

### 打包报错：`pyinstaller==6.3.0` 安装失败

原因：该版本不支持 Python 3.13。

解决：`requirements.txt` 中已更新为 `pyinstaller==6.11.1`，重新安装即可。

---

### Windows 上双击 .exe 一闪而过

原因：程序启动时发生异常，控制台窗口立即关闭。

排查：
1. 在命令提示符中运行 `ExamClient.exe`，查看错误输出
2. 检查 `%APPDATA%\ExamClient\app.log` 日志文件

## 二次开发指引

### 添加新页面

1. 在 `ui/` 下新建 `xxx_frame.py`，继承 `BaseFrame`
2. 实现 `_build()` 构建 UI，实现 `on_show()` 刷新数据
3. 在 `ui/app.py` 的 `_register_frames()` 中注册：
   ```python
   from ui.xxx_frame import XxxFrame
   frame_classes['xxx'] = XxxFrame
   ```
4. 在 `_build_nav()` 的 `nav_items` 列表中添加导航项

### 添加新题型

1. 在 `importer/base_importer.py` 的 `VALID_TYPES` 集合中添加新类型标识符
2. 在 `importer/xlsx_importer.py` 和 `docx_importer.py` 的 `TYPE_MAP` 中添加映射
3. 在 `ui/widgets/question_widget.py` 中添加新题型的渲染逻辑

### 修改 UI 颜色/字体

所有颜色和字体常量集中在 `config.py`，修改对应常量即可全局生效。

### 数据库迁移

如需修改数据库结构（添加列、新建表），在 `database/db_manager.py` 的 `_create_tables()` 中添加 `ALTER TABLE` 或新的 `CREATE TABLE IF NOT EXISTS` 语句。注意保持向后兼容，避免破坏已有数据。
